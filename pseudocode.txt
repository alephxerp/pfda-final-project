screen          : screen
viewport        : basically reads rendergroup, allows for scaling

element         : a thing in the game, button in ui, etc.
processgroup    : holds spritegroups of elements being processed (switches between game, ui, etc.)
rendergroup     : holds spritegroups being drawn to the screen (allows overlay of game, ui, etc)


LOOP:
    update processgroup
        iterates through and checks if mouse is hovering/clicking elements
        if so, calls the element's on_hover or on_click method

    update rendergroup
    draw rendergroup to viewport

    draw viewport to screen

ELEMENTS:
    position, sprite, etc.

    on_hover (makes outline if mouse hovering)
    on_click (does something if mouse clicks)

    update (draw to screen)
